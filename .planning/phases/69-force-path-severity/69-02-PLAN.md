<objective>
Wire two producer-side severity stamps that change no row bytes by themselves
(metadata-only, asserted via the catalog-uat `expectedSeverity` field and -- for
the bulk path -- the summary-line tally):

- SEV-01 missing-companion warning: an otherwise-successful install/update whose
  DECLARED soft-dependency companion (`pi-subagents` / `pi-mcp-adapter`) is not
  loaded renders at **warning** instead of info. Also lock the already-live
  SEV-01 stamps (force degrade -> info; reinstall manual-recovery -> warning) as
  regression guards.
- SEV-04 targeted-vs-bulk decline (D-69-02): a targeted
  `update <plugin>@<marketplace>` declining a force-upgradable upgrade (no
  `--force`, reason `no longer installable`) renders at **warning**; a bulk /
  untargeted `update` skipping one renders at **info**. Thread the EXISTING
  `cardinality` invocation-shape signal -- no new detection, no cascade-shape
  inference.

Purpose: SEV-01 + SEV-04 -- desired-state severity: an explicit force opt-in is
benign (info), a silent degradation of a clean install (missing companion) and a
targeted decline are actionable (warning), a bulk skip the user did not target is
benign (info).
Output: the two conditioned stamps plus catalog-uat `expectedSeverity` updates
and the bulk summary-line tally reconcile.
</objective>
