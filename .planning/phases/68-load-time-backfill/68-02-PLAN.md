<objective>
Widen the Phase 67 unconditional reinstall primitive from installable-only to the
force-capable shape so backfill can reuse it as the single sanctioned re-materialize path
(D-68-02). As built, `reinstallPlugin` resolves through `requireInstallable` (which THROWS
`PluginShapeError {not-installable}` on an `unsupported` plugin) and `updateStateRecord`
hardcodes `compatibility.installable: true` with an always-empty `unsupported` set — so it
can neither run on a force-installed plugin nor record a partial (still-degraded)
re-materialization. This plan swaps the gate to `requireForceInstallable`
(`MaterializablePlugin`), re-types the threaded `installable` fields, and records the REAL
supported/unsupported arrays.

Purpose: This is the BLOCKING prerequisite for the backfill scan (plan 68-04, D-68-02).
Without it, backfilling a still-partial plugin throws and a partial promotion cannot be
persisted.
Output: A force-capable reinstall primitive plus updated/added reinstall tests, including
the side-effect that the standalone `reinstall` command now succeeds on a force-installed
plugin (a desirable repair-primitive expansion, RINST-01).
</objective>
