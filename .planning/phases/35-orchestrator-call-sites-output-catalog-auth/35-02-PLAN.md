<objective>
Wire the GitHub Device Flow into the `marketplace update` orchestrator's
`refreshRecord` helper so a private-GitHub marketplace update flows through
the same auth seam as `marketplace add` (Plan 35-01). On the second-run
case (post-add, credential already in keychain) `credentialOps.fill`
returns the stored token, the closure's `onAuthRequired` is NOT invoked,
and Device Flow does NOT prompt the user -- this is the AUTH-02 silent
reuse contract.

The closure is local to `refreshRecord` (constructed per-refresh) and
forwarded as the 5th positional argument of `refreshGitHubClone` (the
optional `auth?: GitAuthBundle` parameter Phase 34 added). Following the
Phase 34 review IN-02 hint, we KEEP the positional form here (no
options-bag refactor) because `refreshRecord` is the only second caller
of `refreshGitHubClone`; an options-bag refactor at this scale would
require touching both `update.ts` files (marketplace + plugin) and is
deferred to a future cleanup if more callers appear.

Purpose: AUTH-02 (silent token reuse on `marketplace update`). The
end-to-end contract: `marketplace add owner/private-repo` runs Device Flow
once and `credentialOps.approve` persists the token; subsequent
`marketplace update <name>` consults `credentialOps.fill(host)`, gets the
stored token, returns it from `onAuth`, and isomorphic-git's fetch
succeeds with no user-visible auth prompt.

Output:
- `orchestrators/marketplace/update.ts`: `UpdateMarketplaceOptions` and
  `UpdateAllMarketplacesOptions` each gain `credentialOps?: CredentialOps`
  (defaults to `DEFAULT_CREDENTIAL_OPS`) and `deviceFlowHttp?:
  DeviceFlowHttp` (test seam). `refreshRecord` builds the auth closure
  inline for github sources and forwards it as the 5th arg of
  `refreshGitHubClone`. Path sources never reach the auth seam (NFR-5).
- `tests/orchestrators/marketplace/update.test.ts`: at least 2 new tests
  covering (a) AUTH-02 silent reuse end-to-end -- a pre-seeded
  `credentialOps` store yields zero Device Flow notifications; (b) the
  auth bundle is forwarded by reference into the recorded
  `gitOps.fetch.auth` field.
</objective>
