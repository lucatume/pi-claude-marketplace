# Phase 65.1 Plan 02: Reload-Deferred Will Grammar Retirement Summary

the pending `will` prefix means EXACTLY "deferred to the next /reload". The
whole byte-contract change -- closed-set tuples, type union, render arms,
projection, planner DTO, every coupled test, and both prose docs -- landed as
ONE atomic, green commit (D-65.1-04).

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing critical functionality] isReconcilePlanListEmpty must treat immediate-only marketplace actions as non-pending**
- **Found during:** Task 2 (rewriting pending.test.ts MIG-01 + mixed-ordering)
- **Issue:** With `marketplacesToAdd` no longer producing a projection block and
  an empty `marketplacesToRemove` producing none, an immediate-only plan made
  `isReconcilePlanListEmpty` return `false` while the projection returned an
  empty marketplaces array. `pendingReconcile` would then render the
  `(no marketplaces)` sentinel instead of the empty advisory -- violating
  WILL-01 / D-65.1-02 ("a change consisting only of immediate actions yields the
  empty form").
- **Fix:** `isReconcilePlanListEmpty` now drops `marketplacesToAdd` from the
  emptiness check entirely and counts a `marketplacesToRemove` entry only when it
  carries recorded plugins (`m.plugins.length > 0`).
- **Files modified:** orchestrators/reconcile/notify.ts
- **Commit:** 5e102920

**2. [Rule 3 - Blocking] Empty-remove guard in the projection**
- **Found during:** Task 1 / Task 2
- **Issue:** `ensureMarketplaceBlock` in the remove loop would create a bare
  header block even for a remove with zero recorded plugins, contradicting
  D-65.1-03 ("an empty remove shows nothing pending").
- **Fix:** the remove cascade returns early when `o.plugins.length === 0`.
- **Files modified:** orchestrators/reconcile/notify.ts
- **Commit:** 5e102920

**3. [Rule 3 - Blocking] Extracted pushMarketplaceRemoveCascade helper for cognitive complexity**
- **Found during:** Task 3 green gate (eslint)
- **Issue:** The nested remove loop pushed `buildReconcilePendingNotification`
  to sonarjs cognitive-complexity 19 (ceiling 15), failing `npm run lint`.
- **Fix:** extracted the empty-guard + per-plugin push into a module-level
  `pushMarketplaceRemoveCascade` helper.
- **Files modified:** orchestrators/reconcile/notify.ts
- **Commit:** 5e102920

**4. [Rule 1 - Test/comment accuracy] Rewrote tests and comments that depended on the retired tokens**
- **Found during:** Task 2
- **Issue:** The `notify.test.ts` ordering tests built blocks via
  `marketplacesToAdd` (which no longer produces a block); the `pending.test.ts`
  mixed-ordering and MIG-01 tests asserted `(will add)` rows; comments in
  `plan.ts` and `notify.ts` referenced the retired `"will remove"` / `(will add)`
  tokens.
- **Fix:** ordering tests now produce blocks via `pluginsToInstall`; the
  pending tests assert the bare-header / empty-advisory forms; stale comments
  reworded to the immediate/cascade grammar.
- **Files modified:** tests/orchestrators/reconcile/notify.test.ts,
  tests/orchestrators/reconcile/pending.test.ts,
  extensions/pi-claude-marketplace/orchestrators/reconcile/plan.ts,
  extensions/pi-claude-marketplace/shared/notify.ts
- **Commit:** 5e102920
