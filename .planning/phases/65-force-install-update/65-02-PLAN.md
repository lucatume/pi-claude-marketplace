<objective>
Wire `--force` onto the install command so an `unsupported` plugin DEGRADES
(installs the supported components, skips the unsupported ones) instead of blocking,
while `--force` on a fully-supported plugin is inert and `--force` never bypasses
hard failures. No force path emits a `Warning:` summary.

The only force-specific orchestrator difference is GATE SELECTION (D-65-02/03): the
existing single materialize path is reused unchanged because the resolver only ever
puts supported component kinds into `componentPaths`, so the `unsupported` arm skips
the unsupported components naturally with NO separate degrade branch.

Purpose: Deliver FORCE-01 (degrade + no-op), FORCE-03 (block without force),
FORCE-04 (no Warning), and the install half of FORCE-05 (force still blocks
unavailable). Output: force-gated install orchestrator + handler + integration and
parse tests.
</objective>
