<objective>
Wire `--force` onto the update command so that when a plugin's RESOLVED CANDIDATE
(target/newer) version became `unsupported`, the update proceeds by degrading the
now-unsupported components instead of failing (D-65-04). The gate is applied to the
no-network candidate resolve at `preflightUpdate`; without `--force` the candidate
still blocks as `(skipped) {no longer installable}`. `--force` never admits an
`unavailable`/structural candidate, and no force path emits a `Warning:` summary.

Mirrors the install force path (65-02) but against the candidate resolve. Reuses the
single materialize path unchanged (D-65-02) — gate selection is the only
force-specific difference.

Purpose: Deliver FORCE-02 (candidate degrade), FORCE-03 (block without force on
update), FORCE-04 (no Warning), and the update half of FORCE-05. Output: force-gated
update orchestrator + handler + integration and parse tests.
</objective>
