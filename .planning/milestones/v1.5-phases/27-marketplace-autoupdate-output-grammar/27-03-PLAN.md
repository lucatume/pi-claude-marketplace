<objective>
Replace the `(autoupdate enabled)` / `(autoupdate disabled)` status tokens and the `(skipped) {already enabled}` / `{already disabled}` idempotent forms on the `marketplace autoupdate` / `noautoupdate` FLIP surface with marker tokens (UXG-04, closes UAT Finding 4):

- fresh enable → `● <mp> [<scope>] <autoupdate>`
- fresh disable → `● <mp> [<scope>] <no autoupdate>`  (introduces the explicit off-marker; today "off" is marker absence)
- idempotent enable → `● <mp> [<scope>] <autoupdate> {already autoupdate}`
- idempotent disable → `● <mp> [<scope>] <no autoupdate> {already no autoupdate}`

This achieves byte-form parity with the `marketplace list` surface markers while keeping `MARKETPLACE_STATUSES` intact.

DECISION (record in SUMMARY for operator plan-review): **Strategy B** -- keep the existing `MARKETPLACE_STATUSES` discriminators (`autoupdate enabled` / `autoupdate disabled` / `skipped`) and rewrite the renderer ARM BODIES + rename two `REASONS` members. (Strategy A -- routing flips through the list-surface arm -- was rejected: it cannot express the idempotent `{already …}` brace and would shrink `MARKETPLACE_STATUSES`.)

Purpose: This is a multi-file lockstep change (renderer + catalog + byte gate + per-variant test + orchestrator payload + orchestrator test) landing in ONE commit so catalog-uat never sees an intermediate RED. It also INVERTS the `autoupdate.ts` header comment (L30-35) which currently forbids the marker-as-outcome form, and reconciles the catalog's three "`<no autoupdate>` is not emitted" prose statements surface-precisely.
