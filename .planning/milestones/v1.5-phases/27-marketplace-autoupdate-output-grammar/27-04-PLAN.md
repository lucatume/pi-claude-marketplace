<objective>
Make `/claude:plugin marketplace update <name>` render the no-change case as `● <mp> [<scope>] (skipped) {up-to-date}` instead of the always-emitted `(updated)` (UXG-05, closes UAT Finding 5), mirroring the plugin-level up-to-date no-op. Today the autoupdate-OFF manifest-refresh path (update.ts:616-620) unconditionally emits `status:"updated", plugins:[]` even when nothing changed -- there is no "did anything actually change?" signal.

DECISION (record in SUMMARY for operator plan-review): change detection uses **manifest content-compare** (not git SHA, not `lastUpdatedAt`). Research-recommended: content-compare is source-kind-uniform (works for path sources, which never advance a git SHA and are *always* no-ops unless the local file changed) and needs no git-layer plumbing. `record.lastUpdatedAt` is the WRONG signal -- it is stamped to `now` on every refresh regardless of content (Pitfall 4). The compare reads the parsed, typebox-validated `MarketplaceManifest` from `loadMarketplaceManifest` pre/post refresh; equal ⇒ skipped, differ ⇒ updated.

Severity note: `status:"skipped"` routes WARNING via `computeSeverity` (notify.ts:1066-1073). This is intentional for Phase 27 -- do NOT soften benign-skip severity to `info` here; that is UXG-02 in Phase 28 (27-RESEARCH § "Anti-patterns to avoid" -- do not pre-empt UXG-02). The no-op already emits NO `/reload` (plugins:[] ⇒ shouldEmitReloadHint false); UXG-05 is orthogonal to reload-hint.

Purpose: A lockstep change across the orchestrator change-detector + catalog + byte gate + per-variant test + orchestrator test, in ONE commit. The renderer needs NO change -- the existing `renderMpHeader` `skipped` arm already emits `(skipped) {<reason>}` and `up-to-date` is already a REASONS member. Only the orchestrator decision + new/repurposed catalog states are new.

Output: no-op update renders `(skipped) {up-to-date}` (warning, no trailer); changed update still `(updated)`; uniform across both source kinds; all byte surfaces GREEN.
</objective>
