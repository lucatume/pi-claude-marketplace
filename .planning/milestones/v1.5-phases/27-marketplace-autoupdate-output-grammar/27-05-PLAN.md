<objective>
Close the single diagnosed UAT gap in Phase 27 (UXG-05, Test 3, severity major): `marketplace update` on an **autoupdate-ON** marketplace always renders `● <mp> [<scope>] (updated)`, even on a true no-op, because UXG-05's `(skipped) {up-to-date}` rendering was wired only into the autoupdate-OFF branch (update.ts:683-695). The autoupdate-ON branch (update.ts:705-714) unconditionally emits marketplace `status: "updated"` and never consults `snapshot.changed`.

The change-detector logic is correct (proven by the debug session's end-to-end reproduction); it is simply not consulted on the autoupdate-ON path. This plan threads the existing `snapshot.changed` signal plus the cascade outcomes' no-op-ness into the autoupdate-ON branch so a true no-op renders `(skipped) {up-to-date}`, mirroring the autoupdate-OFF no-op.

Folds in the two same-file robustness findings from 27-REVIEW.md (the diagnosis intersects both) and closes the WR-03 coverage gap:
- WR-01: correct the false "typebox `.Parse` yields a stable key order" comments (update.ts:255-256, :301-306). LOCKED DECISION: fix the comment (lowest risk; the detector is byte-stable for same input per the diagnosis), do NOT canonicalize the key.
- WR-02: narrow the bare `catch { return undefined }` in `manifestContentKey` so only ENOENT maps to the changed-safe default; let EACCES / malformed-JSON / schema-invalid PRE failures propagate to the existing `(failed)` routing.
- WR-03: add an orchestrator test that exercises the autoupdate-ON path with a true no-op AND one where a plugin updates, plus catalog-uat + notify-v2 byte coverage for the autoupdate-ON no-op.

Purpose: a no-op `marketplace update` on an autoupdate-ON marketplace (e.g. `claude-plugins-official`) must surface as `(skipped) {up-to-date}`, not the misleading `(updated)` (Phase 27 SC-3 / UXG-05).
