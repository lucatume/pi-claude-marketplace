  <action>
For every fenced output block in `docs/output-catalog.md` that corresponds to a catalog-uat fixture with `expectedSeverity: "error"` or `"warning"`, prepend the summary line to the block content.

The summary line is the first line of the block; it is followed by a blank line, then the existing cascade body. For example, if the existing block is:

```
⊘ claude-plugins-official [user] (failed)
  ⊘ commit-commands (failed) {not in manifest}
    cause: Plugin "commit-commands" not found in marketplace "claude-plugins-official".
```

The updated block becomes:

```
1 plugin operation failed.

⊘ claude-plugins-official [user] (failed)
  ⊘ commit-commands (failed) {not in manifest}
    cause: Plugin "commit-commands" not found in marketplace "claude-plugins-official".
```

The summary line is computed per D-29-03/04: count failed plugin rows (for error) or actionable-skip + manual-recovery rows (for warning) in the catalog example's fixture payload, count failed/skipped-non-benign marketplace rows, then apply the singular/plural and mixed-type grammar.

Catalog blocks for info severity (no `expectedSeverity`) are NOT modified.

Also update the severity-routing commentary near the affected catalog sections (per D-29-07): where prose explains severity computation, add a sentence noting that `error`/`warning` cascades are prefixed with a summary line that counts the failed/skipped operations.
  </action>
