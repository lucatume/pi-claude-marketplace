  <action>
Add a file-private function `buildSummaryLine(message: NotificationMessage, severity: "error" | "warning"): string` co-located with `computeSeverity` in `shared/notify.ts`.

The function performs the same traversal as `computeSeverity` to count operations:

For `severity === "error"`: count plugin rows with `p.status === "failed"` (sum across all marketplaces) and marketplace rows with `mp.status === "failed"`. These are the failed counts.

For `severity === "warning"`: count actionable skips -- plugin rows with `p.status === "skipped"` where `allBenign(p.reasons)` is false, plus plugin rows with `p.status === "manual recovery"`, plus marketplace rows with `mp.status === "skipped"` where `allBenign(mp.reasons)` is false. These are the skipped/actionable counts.

Summary line format (per D-29-03): single counts use singular `"operation"`, plural counts use `"operations"`. Mixed plugin+marketplace uses `"N plugin operation(s) and M marketplace operation(s) [verb]."` sentence. Single type (only plugins or only marketplaces nonzero) uses `"N plugin operation(s) [verb]."` or `"N marketplace operation(s) [verb]."`. Verb is `"failed"` for error severity, `"skipped"` for warning severity.

Example forms:
- 1 plugin, 0 marketplace, error: `"1 plugin operation failed."`
- 2 plugin, 0 marketplace, error: `"2 plugin operations failed."`
- 1 plugin, 1 marketplace, error: `"1 plugin operation and 1 marketplace operation failed."`
- 2 plugin, 1 marketplace, warning: `"2 plugin operations and 1 marketplace operation skipped."`
- 0 plugin, 2 marketplace, warning: `"2 marketplace operations skipped."`

In `notify()`, after computing `severity` via `computeSeverity(message)` and building `withHint` (the current `body + optional reload-hint` string), add the summary composition step:

If `severity === "error" || severity === "warning"`: prepend the summary line. The composed string passed to `ctx.ui.notify` becomes `buildSummaryLine(message, severity) + "\n\n" + withHint`. The `severity` arg is still passed as before (per D-29-01: `computeSeverity` stays active, severity arg preserved).

If `severity === undefined` (info): `ctx.ui.notify(withHint)` -- unchanged from current behavior.

Do NOT modify `notifyUsageError()` -- it is out of scope (per CONTEXT.md domain section).

Add a JSDoc comment on `buildSummaryLine` referencing D-29-02/03/04.
