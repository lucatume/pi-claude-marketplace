<objective>
Refine the v2 first-match severity ladder (`computeSeverity` at `notify.ts:1079`)
so a cascade whose ONLY non-success rows are benign idempotent no-op skips
computes `info` (omit the 2nd `ctx.ui.notify` arg) instead of `warning`.
Actionable skips, manual-recovery, and failures keep their current severity.
This closes UXG-02 (UAT Finding 2) and the Plan 27-04 deferral (mp-level
`(skipped) {up-to-date}` was explicitly left at `warning` "for Phase 28").

Purpose: severity should reflect whether the operator needs to act. A
`marketplace update` no-op or an idempotent autoupdate flip is a benign
"already in requested state" outcome -- warning-coloring it is noise.

Output: a `BENIGN_REASONS` closed set + a rewritten 5-arm `computeSeverity`,
two test gates moved in lockstep (catalog-uat byte gate stays GREEN on strings;
only severity metadata moves), and the ADR / style-guide / catalog severity
prose synced. This is a PURE severity-arg change -- NO rendered byte string
changes.
</objective>
