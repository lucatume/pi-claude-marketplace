<objective>
Wire the GitHub Device Flow into the `marketplace add` orchestrator. When
`marketplace add` is run against a private GitHub repository, `gitOps.clone`
will receive a 401; isomorphic-git invokes the orchestrator-provided
`onAuth` (built by Phase 33's `buildAuthCallbacks` inside `platform/git.ts`)
which consults `credentialOps.fill(host)` first (silent reuse) and, on a
miss, invokes the orchestrator's `onAuthRequired` closure. Phase 35 Plan 01
ships that closure: a function bound to `ctx`, `credentialOps`, and `host`
that delegates to `initiateDeviceFlow` from `domain/github-auth.ts`.

The closure construction is local to `addGithubInGuard` -- it does not
escape into `shared.ts` -- because `host` is derived per-source from the
parsed `GitHubSource` and the closure's `notifyFn` capture is
`ctx.ui.notify.bind(ctx)`. The closure type is exactly `OnAuthRequiredFn`
(declared in `platform/git.ts`), so TypeScript's structural typing accepts
`initiateDeviceFlow`'s `DeviceFlowResult` return shape with no adapter.

Purpose: AUTH-01 (first-access Device Flow on a private GitHub
marketplace). Phase 34 widened `GitOps.clone` with `auth?: GitAuthBundle`;
Phase 35 Plan 01 constructs the bundle and passes it through. The Device
Flow `notifyFn` emits `Open <verification_uri> and enter: <user_code>`
exactly once via `ctx.ui.notify(..., "info")` per the Phase 32
implementation at `domain/github-auth.ts:385`. AUTH-09 holds because the
notify call site interpolates ONLY `user_code` + `verification_uri` (the
access token is not yet in scope when notify fires).

Output:
- `orchestrators/marketplace/add.ts`: `AddMarketplaceOptions` gains
  `credentialOps?: CredentialOps` (defaults to `DEFAULT_CREDENTIAL_OPS`);
  `addGithubInGuard` constructs the `auth` bundle and forwards it into
  `gitOps.clone`.
- `tests/orchestrators/marketplace/add.test.ts`: at least 3 new tests
  covering (a) fill-hit silent reuse (Device Flow NOT triggered),
  (b) fill-miss triggers Device Flow which mocks an HTTP success and
  produces a token, (c) the `auth` bundle is forwarded by reference into
  `gitOps.clone`.
</objective>
