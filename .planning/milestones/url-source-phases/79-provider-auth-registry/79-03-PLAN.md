<objective>
Thread the auth bundle into the plugin clone path (D-77-06 → D-79). Add an optional
`auth?` param to `materializePluginClone` and forward it through the install /
update / reinstall clone-cache seams and their probes, building the bundle via
`buildAuthForHost` (from Plan 02) keyed on the extracted host. Add the once-per-host
memo (D-79-02) at command scope, the plugin-side no-provider cause line (D-79-03), and
keep install.ts / reinstall.ts free of the git surface (no-orchestrator-network gate).

Purpose: private git-source plugins install/update/reinstall against
provider-registered hosts; public plugins on any host clone authless; no-provider
hosts fail clean. This completes PROV-02/03/04 on the plugin side.

Output: an auth-carrying `materializePluginClone`, auth-threaded install/update/
reinstall probes, the command-scope memo, and a new plugin-install auth test.
</objective>
