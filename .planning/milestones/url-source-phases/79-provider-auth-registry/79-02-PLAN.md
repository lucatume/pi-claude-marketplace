<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Clone URL → host extraction | Untrusted URL parsed via `new URL(...).host`; malformed URLs throw before any auth lookup. |
| No-provider host → clone | A host with no registered provider gets NO auth bundle → authless clone → structural fail-clean 401. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-79-04 | Information Disclosure / Spoofing | buildAuthForHost cross-host credential leak | high | mitigate | buildAuthForHost returns undefined when findProviderForHost misses → no bundle for a non-provider host; the bundle's host field is the extracted host so approve/fill key correctly (Pitfall 1). |
| T-79-05 | Denial of Service | infinite auth-retry against a challenging host | high | mitigate | No-provider host attaches NO onAuth → isomorphic-git cannot retry (structural fail-clean, PROV-04). buildAuthCallbacks onAuthFailure always returns { cancel: true } (CP-9, unchanged). |
| T-79-06 | Tampering | wire-format injection via malicious host in credential attribute | medium | mitigate | sanitizeAttrValue in git-credential.ts rejects \r\n\0 in host/username/password (unchanged, now also protects the extracted non-literal host). |
| T-79-07 | Information Disclosure | credential field interpolated into Error/notify in auth-host.ts | high | mitigate | auth-host.ts added to the no-credential-leak gate (Task 3, PROV-05); AUTH-09 discipline. |
| T-79-SC | Tampering | npm/pip/cargo installs | low | accept | No external packages installed this phase (RESEARCH: audit not applicable). |
</threat_model>
