<objective>
Render `url`-sourced marketplaces correctly on the `info` surface (MURL-05) and
map `extraKnownMarketplaces` url entries during `import` (MURL-07). The info work
adds a `url:` projection in `buildBlock`, a `url` arm in the
`MarketplaceInfoMessage.source` union + `renderMarketplaceInfo` switch, and
widens the `last_updated:` gate to all git-backed kinds. The import work upgrades
`marketplaceSourceFromExtra` to read BOTH the current flat shape (no regression)
and the upstream-documented nested `{source: {...}}` shape (adds url + github +
directory).

Purpose: These are the two remaining declarative/display surfaces url sources
must reach. `list` needs no change (D-76-11 â€” list headers carry no source line).
The Open-Q1 `extraKnownMarketplaces` shape was resolved during planning against
the official docs (https://code.claude.com/docs/en/plugin-marketplaces.md,
fetched 2026-07-11): upstream documents the NESTED `{source:{source:"url", url,
ref?}}` shape; the current code reads an undocumented FLAT shape. Reading both is
the no-regression path (D-76-13).

Output: An `info.ts` with a url source projection, a `notify.ts` with a url
render case and a widened last_updated gate, an `import/marketplaces.ts` with a
dual-shape reader, and extended info/import/catalog tests.
</objective>
