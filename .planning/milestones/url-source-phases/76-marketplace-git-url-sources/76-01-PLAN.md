<objective>
Widen the pure source-string parser to accept arbitrary `https://` (non-github)
marketplace sources as the `url` kind, fold `owner/repo@ref` GitHub shorthand
into the `github` kind, make the `samePlannedSource` url arm live and ref-aware,
and mint the `authentication required` closed-set REASONS token. This is the
leaf-layer foundation every Wave-2 orchestrator plan builds on: they need the
parser to emit live `UrlSource` records and the classifier to have a truthful
token for HTTP 401/403 clone challenges.

Purpose: MURL-01 (url parsing), MURL-06 (config reconcile identity for url
sources), and D-76-08 (the new REASONS member) all originate in
`domain/source.ts` and the `shared/notify.ts` REASONS tuple. Isolating them in
Wave 1 lets the two orchestrator plans (add/update/remove and info/import) run
in parallel in Wave 2.

Output: An updated `domain/source.ts` parser, an amended REASONS tuple, a bumped
tripwire assertion, extended domain tests, and catalog/style-guide rows for the
new token.
</objective>

| user CLI string → parser | Untrusted marketplace source string (`https://...`, `owner/repo@ref`) crosses into `parsePluginSource`; syntactic gate only, no I/O |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-76-01 | Tampering | `parseUrlSource` .git/#ref stripping | low | mitigate | Character-level string ops only; no eval, no path resolution. The parsed `url` is handed to isomorphic-git downstream (Plan 02), which enforces `https://`. Parser rejects non-`https://` schemes with truthful reasons (D-76-01). |
| T-76-02 | Spoofing (identity) | github-host normalization (D-76-02) | medium | mitigate | An object-form url pointing at `https://github.com/...` MUST funnel to `github` kind so Device Flow auth still applies; failure to normalize would let a private github repo be cloned unauthenticated. Task 1 adds the funnel; a test asserts `kind: "github"` for the object form. |
| T-76-03 | Information disclosure | `authentication required` reason text | low | accept | The token carries only the HTTP status detail (401/403), never a credential — Phase 76 handles no credentials (public-only). No new file interpolates a token, so the no-credential-leak gate needs no expansion. |
| T-76-SC | Tampering | npm/pip/cargo installs | n/a | accept | This plan installs no packages. `isomorphic-git@1.38.5` is already a committed direct dependency (no new install). No legitimacy checkpoint required. |
</threat_model>
