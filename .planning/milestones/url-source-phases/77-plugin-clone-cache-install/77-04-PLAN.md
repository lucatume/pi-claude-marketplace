<objective>
Wire the clone-cache seam and the sha-version into `install.ts` so a `url` / `git-subdir` / `github`-object plugin installs end to end: the resolver receives a clone-materializing `resolveGitPluginRoot` callback (Plan 02 seam) that calls the Plan 03 clone-cache seam, the version records as `sha-<12hex>` (D-77-01), and the state record persists the full `resolvedSha` (D-77-02). install.ts must remain free of any direct git surface so the `no-orchestrator-network` architecture gate stays green.

Purpose: This is the integration plan. Plans 01-03 built the primitives, the resolver injection contract, and the clone seam. This plan connects them at the install call site, threads the resolved sha through `InstallCtx` into the state record, branches the version derivation for git sources, and updates the architecture-gate rationale comment (the gate's token-grep still passes because install.ts imports the seam by name, never `gitOps`).
Output: extended `orchestrators/plugin/install.ts`, an updated rationale comment in `tests/architecture/no-orchestrator-network.test.ts`, and extended `tests/orchestrators/plugin/install.test.ts` + `tests/architecture/catalog-uat.test.ts`.
</objective>
