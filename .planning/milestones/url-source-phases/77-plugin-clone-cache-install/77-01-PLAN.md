## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user state.json -> cache path | The cache key derives a filesystem path; a malformed/hostile key must never escape `plugin-clones/` |
| persisted version string -> renderer | A malformed pseudo-sha must not be silently rewritten into a misleading short SHA |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-77-01-01 | Tampering | `pluginCloneDir(key)` path composition | high | mitigate | Route the key through `assertSafeName` + `assertPathInside(pluginClonesDir, ...)` (SC-7 chokepoint, Task 2); a `../`/separator key rejects. Test asserts traversal rejection. |
| T-77-01-02 | Tampering | `looksLikeShaVersion` / display transform | low | mitigate | Anchored-exact `SHA_VERSION_RE = /^sha-[0-9a-f]{12}$/` rejects uppercase/wrong-length/affixed strings so a malformed pseudo-sha is never rewritten to a misleading `#<7hex>` (mirrors the existing HASH_VERSION_RE hardening). |
| T-77-01-03 | Information disclosure | cache-key hash truncation collision | low | accept | 12-hex `sha256(url)` truncation matches the locked D-77-04 convention; a collision requires ~2^24 work AND a matching commit sha, and the full url+sha is persisted (D-77-02) for reverse verification. Accepted per the milestone threat model. |
</threat_model>
