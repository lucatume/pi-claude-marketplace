<objective>
Create the fs-only clone garbage-collection helper `garbageCollectPluginClones(locations)`
that both `uninstall` (Plan 04) and `update` (Plan 06) call after their state mutation
commits. It derives the set of still-referenced clone keys from surviving git-source
state records (D-78-01, derive-not-persist) and deletes every `plugin-clones/<key>/`
directory not in that live set.

Purpose: PURL-05 and PURL-06 both require deleting a cached clone once its last
referencing plugin record is gone. This is the single shared primitive that makes both
happen, kept OUTSIDE the orchestrators so `uninstall.ts` imports it without ever touching
the git surface (it stays git-clean; the helper is `readdir` + `rm` only).

Output: `orchestrators/plugin/clone-gc.ts` (new module) + `tests/orchestrators/plugin/clone-gc.test.ts` (new test scaffold — Wave 0 gap).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| readdir(plugin-clones) → rm | Directory names read from disk are untrusted input to a delete operation |
| state.json record → live-key set | Recorded resolvedSource is parser/schema-validated on load; used to protect dirs from deletion |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-78-01 | Tampering | garbageCollectPluginClones rm target | high | mitigate | Every deletion target routes through locations.pluginCloneDir(key) → assertSafeName (rejects /, \, .., control chars) + assertPathInside(pluginClonesDir, candidate) BEFORE rm; a ..-bearing readdir entry is rejected, never deleted (NFR-10) |
| T-78-02 | Denial of Service | shared-clone deletion | high | mitigate | Live-key set derived from ALL surviving records BEFORE any delete (D-78-01); a shared clone stays while any record references it |
| T-78-03 | Availability | partial rm failure strands state | medium | accept | GC runs POST-state-commit (callers' responsibility) and swallows per-dir rm leaks (D-19-01); an orphan is removed by the next idempotent pass (NFR-3) |
</threat_model>
