<objective>
Extract two pure helpers into `clone-cache.ts` so the reinstall (Plan 05) and update (Plan 06)
git-source probes can reuse url reconstruction and git-subdir containment WITHOUT copying code
and WITHOUT reinstall calling `resolvePluginPin` (which would fire a network `resolveRemoteRef`):

1. `canonicalCloneUrl(source)` — the pure url-reconstruction expression currently inline in
   `resolvePluginPin` (clone-cache.ts:136-137). github → `https://github.com/<owner>/<repo>`,
   else `source.url`. No network.
2. `resolveGitSubdirRoot(cloneRoot, subPath)` — the clone-root-anchored containment helper
   currently PRIVATE in install.ts (install.ts:501-528). Moving it to clone-cache.ts lets
   install, reinstall, and update share the exact git-subdir containment logic.

Purpose: PURL-07 requires reinstall to materialize offline from the recorded sha. Reinstall
must NOT call `resolvePluginPin` (it re-resolves unpinned HEAD over the network). It needs only
the canonical url — hence `canonicalCloneUrl`. And its git-subdir arm needs the same containment
install uses — hence the shared `resolveGitSubdirRoot`.

Output: two new exports from clone-cache.ts; install.ts refactored to import them; no behavior change.
</objective>
