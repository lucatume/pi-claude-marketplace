<objective>
Give the reconcile `dangling-reference` diagnostic its own reason token instead of
reusing `source mismatch`. Today a plugin entry whose marketplace is undeclared (e.g.
an orphaned `pr-review-toolkit@claude-plugins-official` local declaration) renders as
`(failed) {source mismatch}` on both the marketplace row and the plugin child —
misattributing an orphaned declaration to a source-comparison problem. Reason tokens
are a closed-set catalog; this is a deliberate catalog amendment.

This is a diagnostic-accuracy fix that complements 78-07 (which removes the orphan at
its source). Even after 78-07 sweeps future orphans, any dangling-reference that arises
(e.g. a hand-edited config) will now name the real problem.

Purpose: When a dangling reference is surfaced, the operator reads
`{dangling reference}` and understands a declared plugin points at a marketplace that
is not declared — actionable — rather than a source-string mismatch that does not exist.

Output: A new `dangling reference` REASONS member (with the partition proof intact),
both render sites switched to it for the dangling-reference cause, and updated tests
proving the divergence from the other three causes.
</objective>
