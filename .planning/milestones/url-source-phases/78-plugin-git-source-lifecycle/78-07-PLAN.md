<objective>
Close the cross-config-layer removal gap: `marketplace remove` and `plugin uninstall`
in standalone mode currently clean the declarative config in exactly ONE physical
layer (base OR local, chosen once via `opts.local`). A plugin declaration for the
removed marketplace/plugin living in the OTHER layer (e.g. `pr-review-toolkit@claude-plugins-official`
in claude-plugins.local.json, installed `--local` by a previous extension version)
survives the remove. The base+local merge has no tombstones, so the leftover key
becomes a dangling-reference on every subsequent `/reload` — "Reconcile: 2 failures"
forever, with no self-heal.

Fix: cascade the config deletion across BOTH layers in the standalone (non-orchestrated)
remove/uninstall paths. Both `claude-plugins.json` and `claude-plugins.local.json` are
inside the NFR-10 sanctioned write set. Orchestrated mode is UNCHANGED (WR-09 protects
per-machine overrides during reconcile-driven derivation).

Purpose: A user who removes a marketplace (or uninstalls a plugin) that was installed
with a previous extension version reconciles cleanly on the very next `/reload` — the
orphaned local declaration is swept as part of the remove, so no dangling-reference
mismatch can persist.

Output: Cross-layer cascade delete in remove.ts + uninstall.ts, plus reproduction tests
that recreate the mixed-layer declaration, run the standalone remove/uninstall, and
assert (a) BOTH physical files no longer declare the key and (b) planReconcile over the
merged config is empty.
</objective>
