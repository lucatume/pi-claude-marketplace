<objective>
Close the MINOR round-2 UAT gap: `marketplace remove` tears down
cascade-uninstalled plugins (bridge unstage + state delete + config cascade)
without ever calling `garbageCollectPluginClones`, so a git-source plugin's
`plugin-clones/<key>/` directory leaks indefinitely when the last referencing
marketplace is removed. `garbageCollectPluginClones` (clone-gc.ts) is already
wired into `plugin/uninstall.ts` and `plugin/update.ts` but not into the
marketplace-remove cascade.

Fix by calling `garbageCollectPluginClones(locations)` in remove.ts's
post-state-commit cleanup block — the same post-commit placement and D-19-01
swallow discipline as uninstall/update — when the cascade fully committed
(failedPlugins.length === 0, i.e. the state delete happened). The GC derives
live keys from the just-committed state, so the removed plugins' clones are
unreferenced and swept, while any clone still referenced by a surviving
marketplace's plugin survives.

Purpose: PURL-06 last-ref GC applies to EVERY uninstall path, including the
marketplace-remove cascade.
Output: one GC call site in remove.ts + a regression test.
</objective>
