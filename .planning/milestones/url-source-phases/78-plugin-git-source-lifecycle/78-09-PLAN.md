<objective>
Close the MAJOR round-2 UAT gap: install completion classifies every git-source
manifest entry as `unavailable` while `list` renders the same entries
`(available)`. The D-78-03 git-source short-circuit and the D-78-04 warm-cache
presence probe were added only to `list.ts`; the completion bucketizer in
`edge-deps.ts` never got either, so `install` completion (which filters
`unavailable` rows out) never offers git-source plugins.

Fix by extracting the two inline probe behaviors from `list.ts` into a shared
probe module colocated with the shared classifier, consuming them in BOTH
`list.ts` and `edge-deps.ts`, bumping the plugin-index cache schemaVersion so
stale wrong-status caches drop+rebuild, and locking parity with an output-parity
drift-guard test that feeds one git-source fixture through both surfaces.

Purpose: one classification for list, completion, install, and update surfaces —
git-source plugins are offered wherever path-source plugins are (PURL-08 / D-78-03 / D-78-04).
Output: shared probe module + wired bucketizer + schema bump + parity test.
</objective>
