<objective>
Add the public `notify(ctx: ExtensionContext, pi: ExtensionAPI, message: NotificationMessage): void` entry point to `extensions/pi-claude-marketplace/shared/notify.ts`. This is the V2 orchestration function consumers will call from Phase 18-20 to emit structured notifications. The function: (1) calls `softDepStatus(pi)` once, (2) iterates `message.marketplaces[]` in caller order (no internal sort -- D-16-06), composing the body via `renderMpHeader` (plan 03) and `renderPluginRow` (plan 04), (3) emits per-plugin cause-chain trailers and `rollbackPartial` child rows at the correct indent levels (D-16-08), (4) joins multi-marketplace blocks with one blank line (D-16-07), (5) appends the reload-hint trailer when state-change criteria are met (D-16-12/D-16-13), (6) computes severity per the D-16-11 ladder and emits via `ctx.ui.notify(body, severity?)` using the V1-established magic-string second-arg convention.

Purpose: SNM-12 names `notify()` the sole public state-change entrypoint of the V2 surface. SNM-14, SNM-15, SNM-16 mandate that severity, reload-hint, and soft-dep probe are computed from contents at notify time. SNM-17 anchors the discriminated-union switch as the sole grammar site (already satisfied by plans 03+04). SNM-18 says the per-variant grammar lives as module-internal helpers, not re-exported composers (already satisfied by 03+04). This plan is the orchestration layer that wires everything Phase 16 ships into a single user-facing function.

Output: Public `notify()` function (~50-80 lines including TSDoc) plus two file-private helpers (`shouldEmitReloadHint(message): boolean` and `computeSeverity(message): "warning" | "error" | undefined`) plus the file-private `RELOAD_HINT_TRAILER` literal, all added to `extensions/pi-claude-marketplace/shared/notify.ts`. The Phase 15 type model, V1 wrappers, V2 `notifyUsageError` (plan 02), and helpers from plans 03+04 stay untouched.
