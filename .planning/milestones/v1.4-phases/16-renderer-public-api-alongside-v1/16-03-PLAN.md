<objective>
Add a file-private `renderMpHeader(mp: MarketplaceNotificationMessage): string` helper to `extensions/pi-claude-marketplace/shared/notify.ts`. The helper is the first of two file-private rendering helpers Phase 16 ships (D-16-09); plan 04 adds the second (`renderPluginRow`). The helper switches on `mp.status` over `"added" | "removed" | "updated" | "failed" | undefined` (5 arms) and returns the v2 marketplace-header line as a single string. Plan 05 will compose this helper into the public `notify()`.

Per the Phase 15 type model (`shared/notify.ts:439-445`), `MarketplaceNotificationMessage.scope` is REQUIRED (so the `[${mp.scope}]` bracket can be emitted unconditionally on every arm), while `MarketplaceNotificationMessage.details?: MarketplaceDetails` is OPTIONAL and INDEPENDENT of `status?` -- the `case undefined:` arm MUST guard `mp.details === undefined` before reading `mp.details.autoupdate` / `mp.details.lastUpdatedAt`, otherwise the read fails TypeScript strict (`Object is possibly undefined`) and crashes at runtime.

Purpose: SNM-17 requires the internal switch over marketplace `status` to be the sole site that knows the v2 marketplace-header grammar. Extracting the helper as file-private satisfies SNM-18's "presentation composers become module-internal helpers" mandate at the marketplace level. The `assertNever` default arm is the compile-time gate that protects against adding new marketplace statuses without updating the renderer (D-16-10).

Output: One new file-private function (~30-60 lines) inside `shared/notify.ts`, plus the small set of inline constants the helper depends on (icon glyphs and the marketplace-header grammar tokens duplicated from `presentation/compact-line.ts` per D-16-04). The Phase 15 type model and V1 wrappers stay untouched. No new exports.
</objective>
