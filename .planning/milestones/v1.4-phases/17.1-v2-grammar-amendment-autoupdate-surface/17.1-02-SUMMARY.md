### Task 2: Severity + reload-hint ladders extended

**File:** `extensions/pi-claude-marketplace/shared/notify.ts`

**`computeSeverity` (now lines 960-985)** -- refactored to two `.some()` chains for cognitive-complexity compliance (D-17.1-02-EXEC-03; see Deviations). The semantic ladder is preserved exactly:

1. First-match: any `mp.status === "failed"` OR any `p.status === "failed"` -> `"error"`. (Unchanged from D-16-11.)
2. Second-match: any `mp.status === "skipped"` OR any `p.status === "skipped" | "manual recovery"` -> `"warning"`. **NEW:** the `mp.status === "skipped"` check at the top of the .some() iterator extends D-16-11 per D-17.1-05.
3. Otherwise: `undefined` (info; omit-2nd-arg per Pi-API convention).

**`shouldEmitReloadHint` (now lines 992-1019)** -- the mp-status check at the top of the outer for-of loop was extended from 3 triggers (`"added"`, `"removed"`, `"updated"`) to 5 triggers (added the two autoupdate flips). `mp.status === "skipped"` is **NOT** added per D-17.1-05 / D-18-05 -- an idempotent no-op does not produce a state change, so the reload-hint suppresses. The plugin-status trigger pass (4 state-changing plugin statuses: installed, updated, reinstalled, uninstalled) is byte-identical.

JSDoc above `shouldEmitReloadHint` was extended with the Phase 17.1 amendment note explicitly calling out the fresh-flip vs. skipped asymmetry.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Test 5 healthy-plugin status changed from "installed" to "available"** (D-17.1-02-EXEC-02)

- **Found during:** Task 3 (writing the mixed-severity test).
- **Issue:** The plan's Test 5 spec required THREE simultaneous assertions:
  - (a) `arguments[1] === "warning"` (severity ladder fires on mp.skipped);
  - (b) body includes `\`● foo [user] (skipped) {already enabled}\``;
  - (c) body does NOT include `\`/reload to pick up changes\``.

  AND specified the plugin row as `status: "installed"`. But per D-16-12 + my Task 2 implementation of `shouldEmitReloadHint`, ANY plugin row with status in `{"installed", "updated", "reinstalled", "uninstalled"}` triggers the reload-hint trailer regardless of mp.status. So assertion (c) would fail given an "installed" plugin row. Task 2 was not authorized to add an "mp.skipped suppresses plugin-row reload-hint contributions" override -- that is a Rule 4 architectural change not specified anywhere.
- **Fix:** Test 5 uses `status: "available"` (a non-state-changing plugin status with no scope/dependencies per MSG-PL-6) for the healthy plugin row. "Available" routes severity to info AND does NOT contribute to the reload-hint ladder, so all three plan-required assertions hold simultaneously. The test's stated purpose (proving "the mp-level routing dominates a non-empty healthy plugin set") is fully preserved; the specific healthy variant is unconstrained as long as it routes to info/no-trigger alone.
- **Files modified:** `tests/shared/notify-v2.test.ts` (Test 5 at line 603).
- **Inline doc:** the test body contains a comment explaining the choice for future readers.

**2. [Rule 1 - Bug] computeSeverity refactored from for-of to .some() chains** (D-17.1-02-EXEC-03)

- **Found during:** Task 2 post-edit `npm run check`.
- **Issue:** Adding the `mp.status === "skipped"` check inside the second-match for-of loop pushed sonarjs/cognitive-complexity from 15 to 16, exceeding the lint gate. The lint error: `Refactor this function to reduce its Cognitive Complexity from 16 to the 15 allowed`.
- **Fix:** rewrote both passes as two `.some()` chains. The new form is semantically identical (same first-match ordering, same return values, same short-circuit behavior) but the cognitive-complexity metric drops below 15 because nested for-loops contribute more weight than chained `.some()` callbacks. No behavior change; the 37 tests still pass.
- **Files modified:** `extensions/pi-claude-marketplace/shared/notify.ts` (`computeSeverity` body at lines 961-985).

**3. [Rule 1 - Bug] Padding-line lint error after new case arm block** (transient; pre-cleanup)

- **Found during:** Task 1 post-edit lint pass.
- **Issue:** `@stylistic/padding-line-between-statements` at line 597 -- the new `case "skipped": { ... }` block ended with a closing brace and was immediately followed by `case undefined: { ...` with no blank line, which the project's flat-config requires.
