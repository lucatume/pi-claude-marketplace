## Accomplishments

- Added Test 4 (`hasLoadedPiMcpAdapter source-substring boundary matches`) -- branch (a). Locks exact-substring match, embedded-in-larger-string match, "pi-mcp" prefix-only negative, empty `source` string, absent `source` field, and non-string `source` (undefined / number 42) -- the last two confirm the `typeof src === "string"` guard at `pi-api.ts:73` protects against runtime crashes.
- Added Test 5 (`probes return false when getAllTools() throws or a tool accessor throws`) -- branch (b). Hardens the existing try/catch fallback coverage; adds an in-callback throw via a `Proxy` whose `name` getter throws, exercising the outer try/catch from inside the `.some()` callback for both probes; also asserts the composed `softDepStatus` clean-negative.
- Added Test 6 (`probes do not crash on tool.name === undefined; fall through to source-substring or false`) -- branch (c), the real coverage gap. Covers: subagent probe returns false for `[{}]` and for `[{ sourceInfo: { source: "pi-subagents" } }]` (subagent probe has no source-substring fallback); mcp probe returns false for `[{}]` and `true` for `[{ sourceInfo: { source: "pi-mcp-adapter" } }]` (source-substring fallback fires when `name` is undefined).
- Widened the file-local `ToolStub` interface to `name?: string` (and `sourceInfo?.source?: unknown`) so the `name === undefined` boundary cases construct without inline casts. Mirrors the `MockTool` shape in `tests/shared/notify-v2.test.ts:144-147`.

## Decisions Made

- **Interface widening over inline casts:** Chose to relax `ToolStub` to `name?: string; sourceInfo?.source?: unknown` rather than spray `as unknown as ...` casts at each test call site. Aligns with `notify-v2.test.ts:144-147` `MockTool` shape.
- **Source field type relaxed to `unknown`:** Plan suggested keeping `string?` but the non-string-source assertion (`source: 42`) required `unknown`. The relaxation is type-safe -- the production probe already does `typeof src === "string"` before `.includes()`.
- **Proxy scoped to throw on `name` only:** Per plan note, both probes still end in `false` because the mcp probe consults `name` first; this preserves test clarity while exercising the in-callback fault path.
