payloads (constructed by orchestrators) and emits strings to `ctx.ui.notify`. There is NO new I/O,
no network surface, no auth boundary, no untrusted-input handling. The only conceptual boundary
crossed is orchestrator → renderer (TypeScript-type-checked at compile time).

| Boundary | Description |
|----------|-------------|
| orchestrator → renderer | TypeScript-validated `NotificationMessage` payload; no runtime parse |
| renderer → `ctx.ui.notify` | string sink owned by Pi API; no escaping required (already string-typed) |
