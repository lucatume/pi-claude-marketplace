<objective>
Reserve the `/reload to pick up changes` trailer for operations that actually
change a Pi-visible resource (skill / agent / command / MCP entry). Collapse the
single chokepoint `shouldEmitReloadHint` to a purely plugin-row-driven rule
(per D-22-01), make the clean `marketplace remove` path carry its `(uninstalled)`
rows (per D-22-02), and drop the now-redundant marketplace-status trigger arms
(including the autoupdate fresh-flip arms per D-22-03). Update the catalog,
catalog-UAT fixtures, and notify-v2 byte/predicate tests in the same change so
byte-equality stays GREEN. Closes SNM-33 (G-MIL-01 / G-MIL-02 / G-MIL-06).

Purpose: A read-only or zero-resource-change marketplace operation must not tell
the user to `/reload`. This mirrors the G-21-01 fix from Plan 21-04 (renderer
byte-identical; reload-gate keyed to a true state-change discriminator).

Output: Modified `shared/notify.ts`, `orchestrators/marketplace/remove.ts`,
`docs/output-catalog.md`, `tests/architecture/catalog-uat.test.ts`,
`tests/shared/notify-v2.test.ts` -- all in one lockstep change.
</objective>
