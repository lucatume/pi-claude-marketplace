<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| persisted version string -> renderer | `formatHashVersionForDisplay` consumes a version string already validated/persisted upstream (by `resolvePluginVersion` and state-IO); it is a pure string-to-string transform with no new external input. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-04 | Tampering | `formatHashVersionForDisplay` output | accept | Renderer-only, pure transform: it strips a prefix and slices 7 hex chars; it cannot alter persisted bytes (`state.json` keeps `hash-<12hex>`, SC#3). The anchored-exact regex `^hash-[0-9a-f]{12}$` means only genuine PI-7 hashes are transformed; any other string passes through verbatim, so no version content is fabricated. |
| T-23-05 | Information disclosure / injection | rendered `v#<7hex>` token | accept | The transform introduces no new injection/escaping surface beyond the existing `v`-prefix discipline -- it only shortens a value already shown to the user; the `#`/hex output contains no control characters or markup. |
| T-23-06 | Spoofing | non-hash strings matching the prefix loosely | mitigate | The predicate is anchored-exact (`^...$`) and lowercase-hex-only, so near-miss strings (uppercase hex, wrong length, `hash-` followed by non-hex) are NOT transformed and render verbatim -- a malformed pseudo-hash cannot be silently rewritten into a misleading short SHA. |
| T-23-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs in this plan; only edits to existing source/test/doc files. No `## Package Legitimacy Audit` required. |
</threat_model>
