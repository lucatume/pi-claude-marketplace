<objective>
Add the `list --unsupported` filter and widen `list --installed` to span the
Phase 66 derived force states, partitioning the list filters cleanly
(LIST-01 / D-67-01).

Purpose: `--unsupported` must select not-installed plugins that resolve
`unsupported` (the force-installable candidates). The blocker is that
`availableRowMessage` collapses resolver `unsupported` and `unavailable` into
the SAME `(unavailable)` render token, so the filter cannot key on `row.status`
alone. The fix threads the internal resolver-state BUCKET to the filter
predicate without changing any rendered byte. In the same change, `--installed`
widens to include `force-installed` + `force-upgradable` (installed-inventory
states), and `--unavailable` narrows to structural-unavailable only so the four
filters partition the surface with no overlap.

Output: orchestrator filter machinery (bucket threading, `shouldShow` arms,
`ListPluginsOptions.unsupported`, `filtersPassive`) plus the edge handler flag
(`BOOLEAN_FLAGS`, token switch, forwarding spread, USAGE); new orchestrator +
handler tests; the completion list-flag-set byte assertion updated; list-filter
prose docs updated (D-67-04).
</objective>
