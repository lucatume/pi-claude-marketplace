<objective>
Extract one shared per-entry plugin-state classifier consumed by BOTH the list
orchestrator and the completion bucketizer, and widen the completion plugin-index
cache to carry the finer derived statuses LIST-02 needs (D-67-02).

Purpose: D-67-02 forbids an independent classifier in the completion provider and
requires the SAME classification `list` uses. Today two classifiers exist:
`installedRowMessage`/`availableRowMessage` in the list orchestrator, and the
`loadManifestForMarketplace` bucketizer in `edge-deps.ts` which collapses
resolver `unsupported`ŌåÆ`unavailable` and marks every state-present plugin as
`installed` (never `upgradable`/`force-*`). This plan extracts the pure
classification core into one helper, refactors list to delegate to it
(behavior-preserving), then widens the cache schema (schemaVersion 1ŌåÆ2) and the
bucketizer so the completion cache carries `installed | upgradable |
force-installed | force-upgradable | available | unsupported | unavailable`. This
is the critical-path foundation for the `--force`-gated candidate sets in 67-04.

Output: a new `plugin-state-classifier.ts` module + its unit test; list
refactored onto it with no behavior change; `completion-cache.ts` schema +
`PluginIndexRow` widened with a version bump; `edge-deps.ts` bucketizer emitting
finer statuses via the shared classifier (no-network); a parity test asserting
the bucketizer and list agree on a shared fixture.
</objective>
