<objective>
Split the resolver's single "hooks failed -> dirty -> unavailable" verdict into two outcomes. `readStandaloneHooks` surfaces `parsed.dropped`; `applyHooksConfig` keeps the structural `!ok -> notes + return true` path (S1/S2/X1 -> unavailable) but, on a successful parse with `dropped.length > 0`, pushes kind `"hooks"` into `partial.unsupported` and threads `partial.droppedHooks`, so `decideResolution` returns `unsupported` automatically. The kept handlers still materialize (push `"hooks"` to `partial.supported` + record `hooksConfigPath`) EXCEPT in the empty-subset case (Q2). `detectOrphanRewake` runs over the filtered subset.

Purpose: This is the verdict change that makes partial-hook plugins force-installable (PHOOK-02) while preserving structural precedence (PHOOK-03). It threads `droppedHooks` for downstream info enumeration (Plan 03) and the byte-exact surface (Plan 04).

Output: `applyHooksConfig` split + `droppedHooks` field/spreads in resolver.ts; new resolver-strict cases.

Locked decisions implemented here:
- D-71-03 (resolver half): supportability -> partial.unsupported; structural -> dirty.
- Q2 (locked as a D-71 sub-decision): empty filtered subset does NOT stage, does NOT set hooksConfigPath, does NOT push to partial.supported; still routes to `unsupported` via droppedHooks (mirrors the LSP-only precedent).
</objective>
