<objective>
De-collapse the resolver `unsupported` arm at the two render collapse points so that a
not-installed, force-installable plugin renders a distinct `(unsupported)` / `ÔŐľ` row in
`list` and `info`, while structurally-`unavailable` plugins keep `(unavailable)` / `ÔŐś`.
This closes the D-64-01 deferral (USTAT-01) and adds the `"unsupported"` closed-set
member with its tripwire bump while leaving the `--unsupported` / `--unavailable` filter
buckets untouched (USTAT-02).

Purpose: hooks/LSP/unsupported-source not-installed plugins become visually distinguishable
from structural failures, completing the three-way render grammar the resolver already
produces.

Output: new `ICON_UNSUPPORTED = "ÔŐľ"` glyph, `PluginUnsupportedMessage` union variant,
`"unsupported"` in `STATUS_TOKENS` (23) and `PLUGIN_STATUSES` (18), split producers in
`list.ts` + `info.ts`, reconciled byte-exact fixtures and `docs/output-catalog.md` /
`docs/messaging-style-guide.md`.

This is a tightly-coupled typed-union change: the closed-set tripwire, the union variant,
the renderer arms, and the two producers must land coherently or `npm run check` breaks.
A single plan / single wave with three ordered, individually-green tasks keeps the tree
compiling and the test suite green after every commit.
</objective>
