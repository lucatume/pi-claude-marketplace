<objective>
Establish the two persisted inputs the load-time backfill version gate depends on:
the running extension version (a checked-in `EXTENSION_VERSION` constant, drift-guarded
against `package.json`) and the stored `lastReconciledExtensionVersion` stamp on
`STATE_SCHEMA`. Per D-68-01 the stamp is an OPTIONAL top-level field with NO
`schemaVersion` bump; an absent stamp means scan-once. The load-bearing hazard closed
here is `loadState`'s normalization, which rebuilds the state object and would silently
drop any new top-level field, leaving the gate permanently open (BFILL-02 / D-68-03).

Purpose: Without these two inputs the backfill scan (plan 68-04) has nothing to compare
or persist; without threading the field through normalization the gate never closes and
backfill re-runs every load.
Output: `shared/extension-version.ts`, the additive `STATE_SCHEMA` field, the
`loadState` normalization fix, and round-trip + drift-guard tests.
</objective>

| T-68-01 | Tampering | state.json optional stamp field | mitigate | `Type.Optional(Type.String())` validated by STATE_VALIDATOR on load and saveState revalidation; a non-string or absent stamp is rejected/ignored, never coerced into the gate. |

| T-68-03 | Denial of Service | loadState on legacy/old doc | accept | Old docs without the field validate unchanged (additive optional, no schemaVersion bump); no migration path that could fail-closed. Local single-user file, low risk. |
