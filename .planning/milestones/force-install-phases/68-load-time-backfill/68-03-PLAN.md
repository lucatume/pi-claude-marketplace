<objective>
Give the load-time reconcile cascade a way to surface backfill promotions WITHOUT adding a
second notification (D-68-04 / RECON-04). Add a `PluginBackfilledOutcome` arm to the
`PerEntryOutcome` union carrying the re-resolved `installable: boolean`, and project it in
`buildReconcileAppliedCascade`: a fully-promoted plugin (unsupported set now empty) renders
as `(installed)` reusing the existing installed row; a partial re-materialize (still
force-installed) renders a `force-installed` row. The reconcile-applied narrow status set
`RECONCILE_APPLIED_STATUSES` (`installed/uninstalled/disabled/failed`) gains a
`force-installed` arm, with its render map and any exhaustiveness/tripwire updated in
lockstep. The `force-installed` token already exists in the global `PLUGIN_STATUSES`
(Phase 66) so this reuses an existing literal; only the reconcile-local narrow set widens.

Purpose: Plan 68-04 produces backfill outcomes; this plan gives them a typed row and a
projection so they fold into the single cascade automatically.
Output: The new outcome arm, the widened applied-status set, the projection arm, and
notify tests. Severity is a sensible default (info) per D-68-04 — Phase 69 finalizes it;
the byte-exact token is frozen in Phase 70.
</objective>
