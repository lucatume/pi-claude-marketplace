<objective>
Wire SEV-03 (D-69-01): the marketplace autoupdate cascade TAKES the force path
for a force-upgradable plugin instead of emitting
`(skipped) {no longer installable}`. The degrading candidate degrades (supported
components materialize, unsupported kinds skip) and renders
`(force-installed) {dropped kinds}`. Severity is conditioned on the PERSISTED
`compatibility.unsupported` record read BEFORE the update applies: empty
(previously clean) means the auto-update NEWLY degrades the plugin ->
**warning**; non-empty (already force-installed) -> **info**.

The manual `update --force` degrade path is unchanged (info; explicit opt-in,
SEV-01). The autoupdate cascade is the SOLE surface where the "newly degrades"
warning fires (RESEARCH A4).

Purpose: SEV-03 -- an automatic upgrade that silently degrades a clean plugin is
actionable (warning); re-degrading an already-degraded plugin is benign (info);
the user never sees a misleading `(skipped) {no longer installable}`.
Output: the autoupdate take-force behavior + prior-state severity conditioning,
with the byte-visible `(skipped)`->`(force-installed)` flip and bulk tally shift
landed in lockstep with `docs/output-catalog.md` + `catalog-uat` fixtures.
</objective>
