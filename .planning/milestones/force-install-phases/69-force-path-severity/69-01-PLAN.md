<objective>
Wire SEV-02: branch the no-`--force` install error on the three-way resolver
state (Phase 64 discriminant) per D-69-03. The `unsupported` arm gets an
error-severity row that carries a clear `--force` hint (force can degrade-install
it); the `unavailable` (structural) arm stays a plain error with NO `--force`
suggestion (force cannot help). This completes the message Phase 65 deferred
(D-65-01).

The structural blocker: `requireInstallable` throws the same
`not-installable` / `no-longer-installable` shape for BOTH `unsupported` and
`unavailable`, so the three-way distinction is lost at the throw and collapsed
to one `(unavailable)` row at the catch. Thread a `forceable` discriminant
through the thrown shape so the catch can append the hint to the unsupported
arm only.

Purpose: SEV-02 -- point the user at `--force` exactly when force can help, and
never when it cannot.
Output: `forceable`-carrying error shape, the conditioned hint trailer, and the
lockstep `docs/output-catalog.md` block + `catalog-uat` fixture so the
byte-equality gate stays green.
</objective>
