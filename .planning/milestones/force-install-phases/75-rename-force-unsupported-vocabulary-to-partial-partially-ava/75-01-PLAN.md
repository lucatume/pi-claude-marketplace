<objective>
Rename the **user-facing flag** and the **internal degrade-force plumbing** from the
`force`/`unsupported` vocabulary to `partial`, WITHOUT touching any user-visible render string,
status/state literal, glyph constant, hint-trailer body, or `shared/notify.ts`. This is the
non-byte-gated half of the Phase 75 rename: it lands as an independently compile-green +
test-green commit whose observable OUTPUT is byte-identical to before (still renders
`(unsupported)`/`(force-installed)`; the failure hint still says `--force`). Plan 02 then flips
the output vocabulary atomically.

This is a pure, behavior-preserving rename (D-75-01). No behavior change, no dependency, no new
command. Apply the RESEARCH §4c symbol-level rule throughout — **no blind substring replace**;
in-scope and out-of-scope `force`/`unsupported` tokens co-occur on the same lines.

Purpose: shrink Plan 02's atomic byte-gate commit by pulling the flag + plumbing rename out into
its own reviewable, always-green commit; keep the delicate `shared/notify.ts` hub out of this plan
entirely so it is touched exactly once (in Plan 02).
Output: `--force`/`--unsupported` retired in favor of `--partial`; the degrade-force option/gate/
status-set-name plumbing renamed to `partial`; all edge + orchestrator + resolver tests updated in
lockstep.
</objective>
