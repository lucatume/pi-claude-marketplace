<objective>
Close the Phase 81 UAT gap: a warm git-subdir plugin (e.g. `canva` in `claude-plugins-official`)
resolves at the monorepo clone root instead of at `<clone>/<source.path>`, so `info` renders
`(available)` with NO components and NO `components: not resolved` marker, and the three-way
verdict can over-claim `(available)` for a subdir whose real components are unsupported.

Root cause (diagnosed against the live codebase in 81-UAT.md): `makePresenceProbe` in
`git-source-probe.ts` returns the bare per-sha clone dir (pinned arm) or the URL-keyed mirror dir
(unpinned arm) as `pluginRoot` and NEVER appends `source.path` for `git-subdir` sources. The
resolver delegates subdir anchoring + containment to the injected callback (D-77-03); the
install-time callback (`clone-cache.ts::resolveGitSubdirRoot`) does `path.resolve(cloneRoot, subPath)`
plus escapes/missing-subdir containment, but the fs-only presence probe skips that tail entirely. The
probe is the single shared seam behind bare `info`, `info --fetch` (via the row-builder probe swap),
`list`, the completion bucketizer, and fetch's post-fetch derived status row -- so all five read
surfaces mis-anchor git-subdir sources (RSTA-04/05 honesty gap). `git-subdir` is the dominant source
kind in `claude-plugins-official`; whole-repo url/github and path sources are unaffected;
install/update/reinstall are unaffected (they already use the clone-cache callback).

Purpose: restore the honest three-way verdict + component enumeration for warm git-subdir plugins
across every fs-only read surface, at parity with the install-time containment, without touching the
network and without growing the closed-set REASONS or status tokens.

Output: an fs-only subdir-anchoring helper shared by the git-tainted clone-cache seam and the
network-free probe; a corrected `makePresenceProbe`; and regression coverage proving warm git-subdir
sources resolve their subdir components and honour the missing-subdir vocabulary.
</objective>
