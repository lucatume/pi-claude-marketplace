# Phase 79.1 Plan 04: Migration/coexistence + GC parity for the mirror shape Summary

**Unpinned reinstall now repairs fs-only from the warm mirror the record points at (no network, no per-sha re-clone, no re-anchor); GC protects mirror-anchored records via the existing first-path-segment derivation while sweeping orphaned per-sha clones — with no persisted migration state anywhere.**

## Accomplishments

- **Reinstall unpinned mirror arm (MIRR-06 / PRL-07).** Added an unpinned arm at the top of `makeReinstallCloneProbe`: when `gitSource.sha === undefined` and a warm mirror dir exists at `plugin-clones/<pluginMirrorKey(cloneUrl)>/`, the probe reads the mirror HEAD fs-only via `readMirrorHeadSha` and returns `{ kind: "materialized", pluginRoot: <mirrorDir or mirrorDir+subdir>, resolvedSha }` — touching no network and no per-sha re-clone. When the mirror dir is absent it falls through to the existing recorded-sha `materializePluginClone` path, so a pre-existing per-sha unpinned clone still repairs (coexistence) and a truly cold source degrades exactly as today (PRL-07: no network materialize on reinstall). Reinstall does NOT re-anchor and does NOT materialize a cold mirror.
- **reinstall.ts stays git-surface-free.** The arm imports only `pluginMirrorKey` (pure), `pathExists` (fs), and `readMirrorHeadSha` (fs-only, git-surface-free) — no `gitOps` / `DEFAULT_GIT_OPS` / `refreshGitHubClone` / `platform/git`. The no-orchestrator-network gate stays green and the git-surface grep returns 0.
- **GC parity verified (MIRR-04).** `clone-gc.ts` is unchanged — RESEARCH confirmed `deriveLiveCloneKeys`' first-path-segment derivation already yields the bare `<urlhash12>` for a mirror record. Added tests proving: a mirror-anchored record's bare 12-hex dir survives GC while an orphaned per-sha clone is swept; a git-subdir mirror record still protects the bare mirror root; an old-design per-sha record still protects its per-sha dir (coexistence).
- **Derive-not-persist proven end-to-end (MIRR-06).** Augmented the existing unpinned update re-anchor test to assert that after the post-swap GC, `plugin-clones/` holds ONLY the bare mirror key — no per-sha orphan and no persisted migration artifact (refcount / stamp). Coexistence derives entirely from `resolvedSource` + on-disk clone presence at read time.

## Decisions Made

- **clone-gc.ts VERIFY-ONLY.** The plan flagged clone-gc.ts as potentially adjustable but RESEARCH-confirmed no change was needed: the first-path-segment derivation protects a bare `<urlhash12>` mirror key exactly like a per-sha key. Verified by the new GC tests + the unchanged diff.
- **Mirror arm at the top of the probe; fall-through preserves the old path.** Placing the unpinned mirror check first and letting the mirror-absent case fall through to the existing recorded-sha path keeps old-design per-sha records repairing and the cold-source degrade unchanged, satisfying PRL-07 (no network on reinstall) with a minimal surgical edit.
- **Augmented rather than duplicated the update test.** The Plan 03 unpinned re-anchor test already asserts re-anchor + mirror survival + per-sha orphan GC; I added the missing derive-not-persist assertion (only the bare mirror key remains) instead of writing a near-identical second test.
