# Phase 79.1 Plan 02: Mirror presence read-surface Summary

Inverted the `makePresenceProbe` unpinned arm from the old `not-cached` degrade
to URL-keyed mirror-dir presence, and added the fs-only HEAD-sha reader the
`materialized` result requires. This is the read-surface half of the mirror
substrate (MIRR-05): read surfaces now resolve an unpinned plugin's components
fs-only from the warm mirror, with zero network and zero git surface.

## What was built

- **`readMirrorHeadSha(mirrorDir)`** (`orchestrators/plugin/git-source-probe.ts`):
  reads `<mirrorDir>/.git/HEAD` fs-only. A symbolic ref (`ref: refs/heads/<b>`)
  resolves from the loose ref file when present, else falls back to
  `.git/packed-refs` (parsed line-by-line, skipping `#` comment and `^`
  peeled-tag lines). A detached HEAD (bare 40-hex sha) returns directly. Uses
  only `node:fs/promises` readFile — it never spawns git nor imports the git
  seam, so the module stays git-surface-free.

- **`makePresenceProbe` unpinned arm rewritten** (same file): when
  `source.sha === undefined`, the probe now computes
  `pluginCloneDir(pluginMirrorKey(cloneUrl))` and, if that mirror dir exists,
  returns `{ kind: "materialized", pluginRoot: mirrorDir, resolvedSha:
  readMirrorHeadSha(mirrorDir) }`; a cold mirror returns `{ kind: "not-cached" }`
  (the arm Phase 80 renders as `(remote)`). The pinned arm is unchanged. Neither
  arm touches the network.

## How it works

The unpinned arm reuses the existing `cloneUrl` reconstruction and `pathExists`
import — it derives the same bare 12-hex key Plan 01's mirror seam materializes
under, so read and write sides agree by construction. The materialized result
supplies the resolver-required non-optional `resolvedSha` by reading the mirror's
checked-out HEAD fs-only. Read surfaces read the warm mirror but never refresh it
(D-79.1-02), so `list`/`info` importers stay network-free (NFR-5).

The HEAD read matches the on-disk layout the clone seam (isomorphic-git in
Plan 01) writes: `readMirrorHeadSha` is verified against a real clone fixture
across loose-ref, packed-refs, and detached-HEAD layouts, asserting exact parity
with `git.resolveRef({ dir, ref: "HEAD" })` (T-79.1-06 / A1).
