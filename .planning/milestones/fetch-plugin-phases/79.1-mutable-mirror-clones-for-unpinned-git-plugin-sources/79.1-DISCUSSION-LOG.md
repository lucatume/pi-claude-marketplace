## Phase origin (re-scope decided during Phase 80 discuss)

The phase was born from the Phase 80 discussion of RSTA-06 (unpinned prefix-scan
ambiguity). Alternatives considered across that exchange:

| Option | Description | Selected |
|--------|-------------|----------|
| Seed rule | Prefix-scan; multi-match = "fetched" without component resolution | |
| Newest-pick heuristic | Resolve from newest clone (fetch mtime or commit date) | |
| Hybrid: mutable mirror for unpinned | One marketplace-style mutable clone per URL; pinned keep per-sha dirs | ✓ |
| Drop sha-addressing entirely | One mutable clone per URL for everything | |

**User's choice:** Re-scope — mutable mirrors first (new Phase 79.1 before 80).
**Notes:** Operator rejected refusing to resolve components when local copies exist
("definitely we don't want to be unable to resolve components of a plugin that we
have information on"). Pins verified as upstream contract (code.claude.com docs:
sha is the effective pin, checked out directly; sha doubles as version) — so
per-sha addressing stays for pinned sources; full drop rejected.

## Re-anchor timing (MIRR-06)

| Option | Description | Selected |
|--------|-------------|----------|
| Lazy, via lifecycle ops | Next update/reinstall re-materializes through the mirror; old per-sha clones stay referenced until then; no migration scan | ✓ |
| Eager load-time backfill | Load-time scan re-anchors all unpinned installs on first run (lastReconciledExtensionVersion-style stamp) | |

**User's choice:** Lazy, via lifecycle ops.
**Notes:** Pure derive-not-persist; no new machinery.

## Marketplace-parity reuse principle (answered refresh-policy and concurrency questions)

Pending questions on refresh policy (which verbs refresh vs read warm) and
concurrency stance (in-place vs staged-swap) were dissolved by the operator's
directive: "can we apply the same semantics and techniques and choices and much
of the same code to these source clones as we do with marketplaces? that way we
wouldn't need to reinvent ideas."

**Resolution:** Mirrors refresh exactly where marketplace clones refresh
(explicit mutating verbs; reads stay warm/fs-only); concurrency and atomicity
inherit `refreshGitHubClone` / staging+rename semantics unchanged; reuse the
actual marketplace helpers where the import graph allows.

**Clarification asked and answered:** "derive-not-persist" — house invariant:
never persist a fact derivable from disk at read time (force-state, GC live-set,
fetched-state precedents).
