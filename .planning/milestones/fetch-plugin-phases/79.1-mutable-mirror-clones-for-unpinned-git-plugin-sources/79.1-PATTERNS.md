## Pattern Assignments

### `domain/clone-key.ts` (domain, transform)

**Analog:** `pluginCloneKey` (same file, lines 31-34). Add a sibling
`pluginMirrorKey` that is the bare LEFT half only — no `-<sha12>` suffix.

**Existing pattern to copy** (lines 20, 31-34):
```typescript
const KEY_TRUNC = 12;

export function pluginCloneKey(canonicalUrl: string, fullSha: string): string {
  const left = createHash("sha256").update(canonicalUrl).digest("hex").slice(0, KEY_TRUNC);
  return `${left}-${fullSha.slice(0, KEY_TRUNC)}`;
}
```

**New symbol** (MIRR-01): `pluginMirrorKey(canonicalUrl)` returns exactly the
`left` expression above (bare 12-hex). Must equal the left half of
`pluginCloneKey(url, anySha)` — assert that equivalence in the test. Reuse the
existing `KEY_TRUNC` constant and the `createHash("sha256")` import already at
line 14; do NOT string-split the two-part key (RESEARCH Alternatives note).

### `orchestrators/plugin/clone-cache.ts` (git-legal seam)

This is the phase's primary work site — the ONLY plugin orchestrator allowed the
git surface. Two patterns combine here.

**Analog A — staging→atomic-rename create path** (`materializePluginClone`, same
file, lines 74-113). The mirror's first-materialization copies this verbatim
EXCEPT: no `checkout({ ref: pin })` (a mirror tracks a moving ref/HEAD, not a
fixed pin), and the key is `pluginMirrorKey(cloneUrl)` not
`pluginCloneKey(url, pin)`.

```typescript
const stagingDir = await args.locations.sourcesStagingDir(randomUUID());
try {
  await gitOps.clone({
    dir: stagingDir,
    url: args.cloneUrl,
    ...(args.ref !== undefined && { ref: args.ref, singleBranch: true }),
    ...(args.auth !== undefined && { auth: args.auth }),
  });
  // MIRROR DELTA: do NOT gitOps.checkout({ ref: pin }) — the working tree tracks HEAD/ref.
} catch (err) {
  const leak = await cleanupStaging(stagingDir, "plugin mirror staging");
  throw appendLeakToError(err, leak);
}
try {
  await mkdir(path.dirname(mirrorRoot), { recursive: true });
  await rename(stagingDir, mirrorRoot);
} catch (err) {
  const code = (err as NodeJS.ErrnoException).code;
  if (code === "EEXIST" || code === "ENOTEMPTY") {
    await cleanupStaging(stagingDir, "plugin mirror staging");
    return mirrorRoot;                 // concurrent win → warm-cache win
  }
  // ... append-leak-rethrow as lines 106-109
}
```

**Analog B — in-place refresh** (`refreshGitHubClone`, `marketplace/shared.ts`
lines 191-260). Reuse VERBATIM; it is host-agnostic and already imported into
this file's dependency graph via `marketplace/shared.ts`. Signature:
`refreshGitHubClone(cloneDir, storedRef, gitOps, onFetchSucceeded?, auth?)`.
`storedRef = source.ref` (branch/tag) or `undefined` (default-branch HEAD
tracking, the D-14 first form, shared.ts 206-224).

```typescript
// Mirror present → refresh in place at marketplace-clone parity (MIRR-02).
let cloneAdvanced = false;
await refreshGitHubClone(
  mirrorDir,
  source.ref,                        // undefined => default HEAD tracking
  gitOps,
  () => { cloneAdvanced = true; },   // flip only AFTER fetch returns (MU-5 pattern)
  authBundle,
);
// Read the checked-out sha for the PURL-09 recorded version (git-legal HERE only):
const pin = await gitOps.resolveRef({ dir: mirrorDir, ref: "HEAD" });
```

**New symbol** (MIRR-01/02/03): `materializeOrRefreshPluginMirror({ locations,
cloneUrl, ref?, auth?, gitOps? })` = `mirrorDir = pluginCloneDir(pluginMirrorKey(cloneUrl))`;
if absent → Analog A; if present → Analog B; both paths end reading HEAD sha
(RESEARCH Open Question 2 recommendation: warm mirror refreshes, does not
skip). Default `gitOps = DEFAULT_GIT_OPS` exactly as `materializePluginClone`
(line 65).

**Existing helpers reused unchanged:** `canonicalCloneUrl` (129-133) for the URL;
`resolveGitSubdirRoot(mirrorRoot, source.path)` (186-213) for the git-subdir arm.

### `orchestrators/plugin/git-source-probe.ts` (fs-only presence probe)

**Analog:** `makePresenceProbe` unpinned arm (same file, lines 71-79). The
current arm returns `not-cached` for an unpinned source (no offline key). MIRR-05
INVERTS this: derive the mirror key and check dir presence, fs-only.

**Current arm to replace** (lines 71-79):
```typescript
if (source.sha === undefined) {
  return { kind: "not-cached" };
}
const key = pluginCloneKey(cloneUrl, source.sha);
const cloneDir = await locations.pluginCloneDir(key);
return (await pathExists(cloneDir))
  ? { kind: "materialized", pluginRoot: cloneDir, resolvedSha: source.sha }
  : { kind: "not-cached" };
```

**New unpinned arm** (MIRR-05): when `source.sha === undefined`, derive
`pluginMirrorKey(cloneUrl)`, `pluginCloneDir(mirrorKey)`, `pathExists`; return
`materialized` from the mirror tree else `not-cached`. Read the sha fs-only —
NEVER `gitOps` (this module has no git import; the no-orchestrator-network gate
and `list`/completion imports depend on it staying git-free). See RESEARCH
Pitfall 3 + Assumption A1: plain-fs read of `<mirrorDir>/.git/HEAD` (verify
loose-vs-packed-ref layout before locking), OR confirm Phase 80 needs only
presence not exact sha. Reuse the existing `pathExists` import (line 32) and the
`cloneUrl` reconstruction already at lines 66-67.
