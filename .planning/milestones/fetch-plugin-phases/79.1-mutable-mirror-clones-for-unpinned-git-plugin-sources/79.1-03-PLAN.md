<objective>
Fork the install and update clone probes so an unpinned git source (`source.sha === undefined`) routes to the mirror seam (`materializeOrRefreshPluginMirror` from Plan 01) instead of the per-sha cache, while the pinned arm is untouched. Both orchestrators are already network-legal and already resolve remote state, so the fork is symmetric. The fork lives INSIDE the clone probe callback — the orchestrator bodies never name the git surface, keeping the no-orchestrator-network gate green.

Purpose: Make install and update consume the mirror working tree and record the resolved HEAD sha for unpinned sources (MIRR-01 write path, MIRR-03), re-anchoring the state record to the bare mirror key so GC protects it (feeds MIRR-04, closed in Plan 04).
Output: Extended install/update seams, forked clone probes, and unit coverage proving unpinned install/update materialize under the bare mirror key with the HEAD-sha recorded version, pinned behavior unchanged.
</objective>
