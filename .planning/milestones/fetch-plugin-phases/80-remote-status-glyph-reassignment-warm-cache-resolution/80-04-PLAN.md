<objective>
Amend the `info` surface (INFO-05) so a not-installed git-source plugin resolves against its warm clone fs-only: a cold clone renders `(remote)` with the existing `components: not resolved` marker, a warm one resolves and lists components via the three-way resolver, and an installed git plugin with a warm clone resolves its components fs-only. Bare `info` stays network-free.

Purpose: RSTA-04/05/06 — "never refuse to resolve components of a plugin we hold local information on" (the 79.1 guiding principle). The current `buildGitNotInstalledRow` unconditionally emits `(available)` + `componentsResolved: false` for every git source; this plan replaces that with presence-derived resolution. Runs parallel with Plan 03 (list surface) — disjoint files.

Output: `info.ts` renders `(remote)`/warm-resolved rows; `buildBlock` threads `locations`; info.test.ts covers remote, warm not-installed three-way, and warm installed component resolution.
</objective>
