## Method

v1.5 exists entirely because of the v1.4 hands-on UAT (`v1.4-MILESTONE-UAT.md`)
-- the six UXG requirements are operator-perceived presentation findings, not
logic bugs. The automated `catalog-uat` byte gate locks the rendered STRINGS but
deliberately does not render COLOR (severity is the 2nd `notify()` arg, not part
of the string). This doc closes that gap in two parts:

1. **Verified-by-capture (deterministic):** the exact body bytes + severity arg
   the extension emits for each surface, captured by running the real `notify()`
   on the canonical payloads. Reproduced from `docs/output-catalog.md` (byte-gated
   GREEN) + a notify-boundary capture harness.
2. **Host render mapping (read from the bundle):** what the host does with each
   severity arg. Authoritative source: the installed host bundle.
3. **Operator-confirm (needs eyes):** the actual on-screen color/label in the live
   TUI -- the part no test or capture can assert. Checklist at the bottom.

## Host severity -> render mapping (authoritative, host v0.75.5)

`interactive-mode.js :: showExtensionNotify(message, type)` dispatches:

| severity arg (2nd notify arg) | host method | render |
| ----------------------------- | ----------- | ------ |
| `"error"` | `showError` | `theme.fg("error", "Error: " + msg)` -- error color **+ `Error: ` label prefix** |
| `"warning"` | `showWarning` | `theme.fg("warning", "Warning: " + msg)` -- warning color **+ `Warning: ` label prefix** |
| omitted / info | `showStatus` | `theme.fg("dim", msg)` -- **dim, NO label prefix** |

This single mapping is the crux of the whole milestone: the only label-free path
(`showStatus`) is also the only NON-severity-colored path (`dim`). That is why
UXG-02 (route benign no-ops to info) WORKS -- benign no-ops now render dim and
label-free -- and simultaneously why UXG-03 (color WITHOUT label on multi-line
cascades) is UPSTREAM-INFEASIBLE: you cannot get the error/warning color without
the `Error:`/`Warning:` label.

## Captured evidence by requirement

### UXG-02 -- benign skip cascades route to `info` (the headline fix)

Captured severity arg per scenario (real `notify()` path):

| scenario | emitted body | severity arg | host render |
| -------- | ------------ | ------------ | ----------- |
| `marketplace update` no-op (autoupdate OFF) | `● local-mp [user] (skipped) {up-to-date}` | **info** (no 2nd arg) | dim, no label |
| `marketplace update` no-op (autoupdate ON, all unchanged) | `● official [user] (skipped) {up-to-date}` | **info** | dim, no label |
| all-plugins-up-to-date cascade | `● local-mp [user]` / `  ⊘ alpha (skipped) {up-to-date}` / `  ⊘ beta (skipped) {up-to-date}` | **info** | dim, no label |
| autoupdate idempotent enable | `● foo [user] <autoupdate> {already autoupdate}` | **info** | dim, no label |
| autoupdate idempotent disable | `● foo [user] <no autoupdate> {already no autoupdate}` | **info** | dim, no label |

Warning contrast (still actionable -> still colored + labeled):

| scenario | emitted body | severity arg | host render |
| -------- | ------------ | ------------ | ----------- |
| actionable skip (`not installed`) | `● demo [user]` / `  ⊘ alpha v1.0.0 (skipped) {not installed}` | **warning** | yellow + `Warning: ` |
| empty `reasons: []` (cannot prove benign) | `● demo [user]` / `  ⊘ alpha v1.0.0 (skipped)` | **warning** | yellow + `Warning: ` |
| manual recovery | `● official [user]` / `  ⊘ helper v1.0.0 (manual recovery) {unreadable}` / `    cause: ...` | **warning** | yellow + `Warning: ` |

Verdict (by capture): **PASS** -- benign-only skip cascades emit info; actionable
/ mixed / manual-recovery emit warning. Body bytes unchanged vs pre-v1.5.

### UXG-03 -- host label on multi-line cascades (DEFER-WITH-FINDING)

This is NOT a shipped fix -- it is a confirmed upstream limitation
(`UXG-03-FINDING.md`, D-28-12). The multi-line error cascade captured:

```text
● official [user] (updated)
  ● alpha 0.5.0 → v1.0.0 (updated)
  ⊘ beta (skipped) {up-to-date}
  ⊘ delta (failed) {network unreachable}

/reload to pick up changes
```

severity arg = **error** -> host `showError` prepends `Error: ` to the WHOLE
block, so the rendered first line becomes `Error: ● official [user] (updated)`
(shifting it off the `0 / 2` indent ladder) and the entire block is error-colored.
There is no host path to color it without the label.

Verdict: **CONFIRMED FINDING (expected, accepted)** -- the label still prepends
on `warning`/`error` multi-line cascades. NOTE the silver lining from UXG-02: the
COMMON multi-line no-op cascades (all-up-to-date) now route to info, so they no
longer get the `Warning:` label -- the formatting-break now only affects genuinely
actionable/failed cascades, where the label arguably belongs.

## Update 2026-05-31 -- runtime pass reopened the milestone (Phase 29)

The deterministic capture above proved the severity-ARG mapping but used
synthetic payloads, so it never exercised the real `install` / `update`
classification path. The operator then ran the source-loaded v0.2.0 runtime
directly and hit three cases the capture missed (commands shown above each
block):

```text
/claude:plugin install beta@uat-mp
 Error: ● uat-mp [user]
   ⊘ beta (failed) {already installed}
     cause: Plugin "beta" is already installed in marketplace "uat-mp".

/claude:plugin update alpha@uat-mp
 Warning: ● uat-mp [user]
   ⊘ alpha (skipped) {not installed}

/claude:plugin update up@uat-mp        # "up" does not exist in the manifest
 Warning: ● uat-mp [user]
   ⊘ up (skipped) {not installed}

/claude:plugin install up@uat-mp       # install gets it right:
Error: ● uat-mp [user]
   ⊘ up (failed) {not in manifest}
     cause: Plugin "up" not found in marketplace "uat-mp".
```

Two findings, now Phase 29:

- **UXG-07** -- the host `Error:`/`Warning:` label still prefixes multi-line
  cascades wherever severity is non-info (cases 1-3), breaking the indent
  ladder. Since the operator has confirmed severity COLOR is expendable, the
  label IS removable in-extension by routing `notify()` cascades through info
  (omit the 2nd arg) -- only "color WITHOUT label" was upstream-infeasible.
  This supersedes the UXG-03 defer-with-finding and retires/dormants the UXG-02
  ladder. Entrypoint-split (D-28-13) keeps the single-line `notifyUsageError`
  label.
- **UXG-08** -- case 3 is also a classification bug: `update up` (nonexistent)
  renders `(skipped) {not installed}` when it should render
  `(failed) {not in manifest}` like `install up` (case 4). Root cause:
  `preflightUpdate` checks the installed-state record before the manifest
  (`orchestrators/plugin/update.ts:592`).

(Possible UXG-09 for discuss: case 1 -- `install` of an already-installed
plugin renders `(failed)`/error; reclassify to a benign no-op?)
