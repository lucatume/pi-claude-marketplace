<objective>
Collapse the three unlinked copies of each CLI verb's flags (handler parse sets,
completion candidates, USAGE strings) into ONE per-verb flag catalog module in the
edge layer, then derive the handler parse sets AND the completion candidate list
from that catalog so drift becomes impossible by construction. Closes the two
confirmed completion-drift gaps: `list --remote` (RSTA-07) and `info --fetch`
(FTCH-03), which the handlers already parse but the completion never offered.

Purpose: today drift is undetectable because the completion tests are
inclusion-only. A new flag added to a handler silently fails to appear in
completion, and vice versa. A catalog + exact-set drift guard makes the two
surfaces provably consistent.

Output: `edge/flag-catalog.ts`; `provider.ts`, `list.ts`, `info.ts` rewired to
read the catalog; a new architecture drift-guard test; and the inclusion-only
completion assertions tightened to exact-set.
</objective>
